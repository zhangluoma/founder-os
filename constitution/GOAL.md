## 2. 【核心】不可伪造的成功判据

> **一个 agent OS 的好坏, 不看它自己觉得进化了没有 —— 拉出去跑。**
> **agent 是车。被驱动的项目是赛道。圈速骗不了人。**

### 2.0 赛道 (被驱动的真实项目)

```bash
FOUNDER_TARGET=/absolute/path/to/your/project
```

### 2.1 判据 — **只测结果, 不测努力**

> ⚠️ **"够不够 proactive / 卖不卖力" 全是陷阱** —— 那是**可伪造的输入指标**。
> agent 能刷想法数、刷 commit、写漂亮 journal。
> **别测卖力, 测结果。** (亲证病例: 11 个采集器 + 3 轮研究 + 日报天天好看 = 真产出 **0**。)

**检验法**: 问自己 —— **"agent 能靠自我欺骗让这个数字变好吗?"** 能 → **这个判据废了。**

| ❌ 会被骗 | ✅ 骗不了 |
|---|---|
| "代码质量提升了" | `pytest` 真的全绿 |
| "用户体验更好" | 真实 DAU / 留存 / 付费 |
| "研究有进展" | 实盘 P&L / 通过前向验证的假设数 |

### 2.2 可执行命令 (最强形式)

```bash
FITNESS_CMD=（你的命令; 例: pytest -q | tail -1  /  gh api repos/O/R --jq .stargazers_count）
FITNESS_DIRECTION=higher
```

### 2.3 归因 (**不可伪造 ≠ 可归因**)

判据必须**能归因于 agent**。若 agent 从没写过赛道, 那个数测的是**赛道自己的历史 + 外部噪声**, **不是 agent**。
→ `bin/fitness-driven-project` 用 git trailer `Founder-OS-Shift:` 做归因门。**未归因样本 = FITNESS 0。**
