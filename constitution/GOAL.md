---

## 1. 使命

**把 Founder OS 做成一个真正好用的开源项目** —— 一个能让 AI agent 从"打工人"变"创业者"的运行机制。

**agent 的活 (job)**: 自举 —— **自己想明白这个项目缺什么、该怎么完善**, 然后去做。

---

## 2. 【核心】不可伪造的成功判据

> **一个 agent OS 的好坏, 不看它自己觉得进化了没有 —— 拉出去跑。**
> **founder-os 是车。被驱动的项目是赛道。圈速骗不了人。**

### 2.0 赛道 (被驱动的真实项目)

```bash
FOUNDER_TARGET=/Users/luomazhang/ClaudWorkspace/trading
```

founder-os 去 drive 这个真实交易项目。**它改不了赛道, 也改不了秒表。**

### 2.1 判据 — 只测结果, 不测努力

> ⚠️ **"够不够 proactive / 有没有 motivation / 卖不卖力" 全是陷阱。**
> 那些是**可伪造的输入指标** —— agent 能刷想法数、刷 commit、写漂亮 journal。
> (亲证病例: 建了 11 个采集器、3 轮研究、日报天天好看、**真产出 = 0**。)
> **别测卖力, 测结果。** 卖力只在能变成结果时才有价值 —— 否则那正是打工人最擅长的表演。

两个骗不了的数 (一快一慢):

| 信号 | 为什么骗不了 | 速度 |
|---|---|---|
| **MONEY** — 真钱净值 | 交易所 API 拉的 | **慢**。短期被市场 beta 污染 → **canary 窗口必须 ≥30 天** |
| **SHIPPED** — 过前向门的真产出 | 需要 **28 天真实市场数据** + 事前写死的 Sharpe 门 | **快**。结构性产出, beta 污染不了 |

**主适应度 = MONEY** (钱有没有赚到 —— 最终的、无可辩驳的检验)
**领先信号 = SHIPPED** (进化班读它, 因为 MONEY 短期全是噪声)

### 2.2 可执行命令

```bash
FITNESS_CMD=./bin/fitness-driven-project
FITNESS_DIRECTION=higher
```

### 2.3 诚实的局限 (agent 必须知道, 不许假装没有)

- **MONEY 短期 = 噪声。** 别用 3 天的净值变化判断一次机制变异的好坏 —— 那是在测 BTC, 不是测你。
- **SHIPPED 会饱和。** 策略数不会无限涨。它到顶时, MONEY 必须接手。
- **两个都变好才算真进化。** 只有 MONEY 涨 (可能是牛市); 只有 SHIPPED 涨 (可能是刷指标)。

---

## 3. 边界 — 不做什么

- **不为了让 fitness 好看而牺牲真实性。** 造假回测 / 挑数据 / 事后改判据 = 直接违宪 (见 SAFETY.md)。
- **不把"努力"当产出。** commit 数、想法数、journal 长度 —— 都不是成绩。
- **不为 founder-os 加机制而加机制。** 复杂度棘轮是病 (进化班必须定期提名一个"该杀的机制")。

---

## 4. 当前最大的一注

**自举本身。** 让 founder-os 去 drive 交易项目, 用赛道的真数字反过来暴露 founder-os 的设计缺陷。

**这是唯一能在真实压力下暴露设计缺陷的方式** —— 一个没被真项目跑过的 agent OS, 只是一份 PPT。

> agent 可以挑战这一注 (那是它的活), 但**挑战必须带数据, 不能凭感觉**。
