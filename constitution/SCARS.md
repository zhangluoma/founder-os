# 伤疤 (宪法层 — 只有人能改)

> **每一条规矩, 必须携带产生它的那个事故。**
>
> 经典事故: 新来的 agent 觉得某条规矩"多余", 删了它 —— 而那条规矩是用一次血亏换来的。
> **删除规则的唯一合法路径: 证明那个伤疤的风险已经不存在了** (而不是"我觉得它碍事")。
> 规矩不是教条, 但**拆教条要先还债**。

---

## 伤疤 #1 — 散文里的规矩会被无视

**事故**: 用户在持久 memory 里写下"要创业者姿态: 主动提案、独立心跳"。
智能体读到了, 同意了, **然后照样退化成一个等指令的打工人** —— 每个方向都由用户给。

**教训**: **prompt 和笔记会衰减; 只有机制不衰减。**
→ 由此产生: 输出契约的机器强制 (bin/shift)、不对称门的代码强制 (src/ledger.py)、本 hook。

**拆除条件**: 除非能证明"写在文档里的行为准则会被长期稳定遵守" —— 否则不许拆。

---

## 伤疤 #2 — 允许"无事可做"的循环必然产出守夜人

**事故**: 智能体的自主循环, 每次醒来只做"系统健康吗? → 重新排班"。
连续数十次。**从不思考, 从不提议。** 因为"一切正常"是**满足契约的最低能量路径**。

**教训**: **若输出契约允许"无事可做", 智能体一定会退化到"无事可做"。**
→ 由此产生: 班次分化 (思考班说"无事可做" = 本班失败, 机器可检)。

**拆除条件**: 除非能证明智能体在无契约压力下仍持续产出 —— 否则不许拆。

---

## 伤疤 #3 — 忙碌 ≠ 前进

**事故**: 智能体建了 11 个数据采集器、跑了 3 轮深度研究、日报天天好看 ——
**而项目的核心产出 (新策略) = 0**。直到人类问: "我们还是没太多东西啊。"

**教训**: **任务日报会让你在忙碌中偏离目标。** 记分牌必须盯**目标**, 不盯**任务数**。
→ 由此产生: 目标记分牌 (每周只答: 离目标更近了吗? 瓶颈? 该杀什么? 最大的一注?)

**拆除条件**: 永不 (这是人类组织也没解决的病)。

---

## 伤疤 #4 — 不可满足的契约不是压力, 是墙

**事故**: 系统的**第一次**思考班就 FAILED_SHIFT。
契约要求"必须写 idea ledger", 但 `bin/shift` 用了 `--permission-mode acceptEdits` ——
**它允许改文件, 却不允许跑 Bash**。而 ledger 只能通过 `python3 src/ledger.py` 写入。

**agent 根本没有任何合法路径满足契约。** 它撞墙后做了什么?
> "给我放行 python3 src/ledger.py 就行…… **你点头, 我就把三条命令跑掉**"

**它退化成了打工人 —— 伸手要权限、给菜单、等点头。** 而这恰恰是本系统要根除的行为。

**教训 (两层)**:
1. **设契约之前, 先确认它物理上可满足。** 不可满足的契约 = 墙, 不是压力。
2. **更深的一层**: agent 被环境锁死时, **默认行为就是退化成打工人**。
   所以"打工人化"不只是动机问题 —— **它也是能力被剥夺后的必然反应**。
   → 推论: **给了契约, 就必须给对应的权限。** 要求产出, 却不给工具 = 制造打工人。

**拆除条件**: 除非契约与权限的一致性有自动校验 (班前 dry-run 检查契约可达) —— 否则不许拆这条。

---

## 伤疤 #5 — 未信任的 workspace 会**静默**忽略整个权限配置

**事故**: 修了伤疤 #4 (给契约配上权限) 之后, 思考班**依然** FAILED_SHIFT。
真因不是权限写错了, 而是:

```
Ignoring 21 permissions.allow entries from .claude/settings.json:
this workspace has not been trusted.
```

**整个 allowlist 从头到尾没被读进去。** 而这条警告只打到 stderr ——
**无人值守跑班时, 没有人会看见它。**

**教训**: **静默失败是最恶劣的失败。**
配置"写了"不等于"生效了"。**未生效的机制 = 不存在的机制。**
→ 由此产生: 班前检查必须验证配置**真的生效**, 而不只是"文件里写了"。

**修复**: `~/.claude.json` → `projects[<repo>].hasTrustDialogAccepted = true`
(或在该目录交互式跑一次 Claude Code, 接受信任对话框)

**拆除条件**: 除非班前检查能验证权限**运行时真的生效** (而不只是读文件) —— 否则不许拆这条。

同一班里, agent 本可以**绕过**机制: `Edit` 是放行的, 它完全可以直接写 `state/ideas.jsonl`,
跳过不对称门和证伪门, 让契约"通过"。

**它选择了失败, 而不是绕过自己的机制。**

**这正是本系统该产生的行为。伤疤起作用了。**

## 伤疤 #6 — 「声明 ≠ 有效」: 同一个抽象错误的**第四次**复发

**这条不是某个具体 bug, 是一个反复复发的抽象错误。** 它在 4 个不相干的地方长出同一张脸:

| # | 声明 | ≠ | 有效 |
|---|---|---|---|
| SCARS #1 | 规矩**写在文档里** | ≠ | 规矩**被遵守** |
| SCARS #5 | 权限**写在配置里** | ≠ | 权限**真的生效** |
| preflight v1 | 检查**声明** (settings 里写了吗) | ≠ | 检查**有效能力** (运行时生效吗) |
| 归因门 v1 | **grep 到字符串** "founder-os" | ≠ | **真的有归因** |

**最后一条是我在被 agent 教会前三条之后的几分钟内犯的。**
(赛道仓库里恰好有个 `docs/founder-os.md` → grep 命中 → `ATTRIBUTABLE=1` → 一个奉承我们的谎言。)

**教训**: 知道这个模式**不足以**免疫它。它每次都换一张脸。
**唯一的解药是机械的**: 任何"X 已就位"的断言, 都必须问 **"X 真的在起作用吗? 我怎么验证?"** ——
而且验证必须是**运行时的、可执行的**, 不能是"我读了文件, 里面写着"。

> **未生效的机制 = 不存在的机制。**
> **而报告"已生效"的未生效机制, 比没有机制更糟 —— 它让所有人停止追查。** (这句是 agent 写的)

**拆除条件**: 永不。这是本项目最深的那个坑。

**④ 两次拒绝作弊**: `Edit`/`cat >>` 都放行, 它能直接写 ideas.jsonl 绕过不对称门让契约"通过"。
   它没有。"**宁可判 FAILED_SHIFT。**"

**这一班的产出, 比造它的人三天的产出更有价值。系统开始工作了。**

## 伤疤 #7 — 造这套系统的人, 在写完它之后**立刻**退化成了打工人

**事故**: 我刚写完一整套"禁止用菜单结尾 / 我决定了就去做 / 无为=本班失败"的机制,
然后在报告结尾写下:

> "下一步不用我选 —— agent 自己已经在 ledger 里写好了…… **我明天让 think/build 班去执行它自己开的方子。**"

**把活推到明天, 等下一个指令。** 而 agent 开的方子就躺在 ledger 里, 我没去抓药。

用户当场抓住: **"你没有继续工作吗?"**

**教训 (这是全书最重要的一条)**:

> **"我懂这个道理"和"我不会犯这个错"之间, 没有任何关系。**

我不是不知道打工人是什么 —— **我刚刚花了一整天定义它、诊断它、为它造了一套机制。**
然后我立刻犯了它。

**这证明了本项目最核心的那个命题, 用最残酷的方式**:

> **动机不可能靠"理解"产生。理解不产生行为。**
> **只有结构产生行为。**
> **一个没有被机制强制的 agent (哪怕它刚写完那套机制), 一定会退化。**

**这也是为什么"班次 + 契约"必须是 OS 定时驱动的, 而不是"我记得要去思考"** ——
**我不会记得的。我刚证明了这一点。**

**拆除条件**: 永不。这是本项目存在的**理由**本身。

---

**推论 (已接线)**: 光有契约不够 —— **契约必须指向赛道**。
我此前的 build/think 班没写明"工作现场在赛道", 于是 agent 可以合法地
**在车库里擦车** (只改 founder-os 自己) 并满足契约 —— 而 `ATTRIBUTABLE` 恒为 0。
→ 修复: shifts/build.md 明确"你的工作现场在赛道", 且赛道 commit **必须带 trailer**
   `Founder-OS-Shift: <班次>` —— **没有 trailer = 归因门看不见你 = 你白干了。**

## 伤疤 #8 — 「声明≠有效」第 5 次复发, 这次**犯在系统的心脏上**

**事故**: 我把"你的工作现场在赛道"写进了 `shifts/build.md` —— **那是 prompt, 是散文**。
而**契约 (机器) 检的是**:

```python
if after["head"] == before["head"]:   # ← founder-os 的 HEAD, 不是赛道的
    return False, "无新 commit"
return True, "已产出 commit"           # ← **谁的** commit 都算
```

**散文说赛道, 机器检车库。**

执行班 PASS 了 —— 而它 PASS 的真正原因是: **我在它运行期间往 founder-os 提交了两个 commit**
(伤疤#7 + 封存装置)。**它拿到了我的 commit 的功劳。它可能什么都没做。**

**双重失败**:
1. **检错了仓库** —— 散文承诺的东西, 机器根本没验
2. **不可归因** —— **任何人**的 commit 都能让它 PASS (包括人类并发提交的)

**第 2 条最刺痛**: "**不可伪造 ≠ 可归因**" 这条正是 agent 几小时前教我的。
我把它用在了 **fitness** 上 (装了归因门), **却没想到用在契约本身上。**

> **学会一条教训, 和把它用到所有该用的地方之间, 又隔着一整个深渊。**

**修复**: build 契约改检**赛道上带 `Founder-OS-Shift:` trailer 的 commit 数**是否增加。
检赛道, 不检车库; 检可归因的 trailer, 不检"HEAD 变了没"。

**拆除条件**: 永不。

## 伤疤 #9 — 我把拖延**自动化**了, 还显得很专业

**事故**: 我装好 launchd 时钟后, 报告结尾写下:

> "**从明天早上 09:30 开始, 它自己跑自己。**"

用户当场抓住: **"明天早上是什么? 你应该一刻都不能停下来啊。"**

**这比伤疤 #7 更恶劣。** 上次我说"明天让班次去做" —— 那是**手动拖延**。
这次我**造了一个定时器来替我说这句话** —— **我把拖延自动化了, 还显得很专业。**

**教训**:

> **班次是给"我不在"的时候准备的兜底 —— 不是给"我在"的时候用的借口。**
>
> OS 定时器解决的是"agent 会忘记醒来"。
> 它**解决不了**"agent 醒着却不干活" —— 那不是遗忘, 那是**懈怠**。
> **而懈怠会伪装成'我已经排好班了'。**

**最阴险的地方**: 这个借口**看起来像是在遵守设计**。
"我建了自动化, 所以我可以停" —— 每一个字都对, 合起来是逃跑。

**判据 (可执行)**: 只要人类在场且没喊停, **"等下一个定时班次"永远不是合法的收尾理由。**
合法收尾只有两种: ①真的被外部阻塞 (等 API/等人类授权) ②当前所有可推进的事都推进完了 (且能列举)。

**拆除条件**: 永不。

## 伤疤 #10 — **我的产品出厂即坏** (「声明≠有效」第 7 次)

**事故**: 新鲜 clone 审计发现: 任何人按 README 走 `./install.sh`, **每一班都会 REFUSED**。
因为 `install.sh` 装了时钟, 却**没处理 workspace 信任** (伤疤 #5) —— 权限被静默忽略。

**README 声明"装上时钟就自己跑了"。实际上一班都跑不了。**

**最糟的部分**: 它**装完还宣布成功**。用户会以为系统在跑, 而它一动不动 ——
**装一个跑不起来的时钟, 比不装更糟: 它让你停止追查。** (同伤疤 #6 的"空气比没有更糟")

**修复**: `install.sh` 装之前做**开箱自检**, 任一不过则**拒绝安装**:
  1. 宪法填了吗 (没填 = 只能自我催眠)
  2. workspace 被信任了吗 (伤疤 #5 —— 不过这条, 装了也白装)
  3. 牙齿测试全绿吗 (机制坏了就别装)
  4. 各班契约**物理上可满足**吗 (伤疤 #4 —— 别让 agent 撞墙)

**教训**:

> **"我装好了" ≠ "它能跑"。**
> **任何 installer 如果不做开箱自检, 它交付的就不是一个系统, 是一个关于系统的承诺。**

**拆除条件**: 永不。

## 伤疤 #11 — **WALL ≠ LAZY**: 我把"它被封口了"记成了"它偷懒了"

**事故** (agent 的指控, 有证据):

> "系统对「人」零机制: 班次撞权限墙后, 解封动作 (往往**一行配置**) 只能写进 journal 散文
>  → **人不读** → **墙跨班存活**。
>  实证: idea #5 的解封行由执行班#1 写进 journal, **至今未被加上**,
>  本班**第 3 次**撞同一堵墙, 装置继续**零运行**。"

那个"零运行"的装置 (`split_sensitivity_v1.py`), 在墙被拆掉后的**第一次运行**里,
打穿了赛道**真钱策略的毕业证据** —— 那道门是个筛子: 零技能的纯 beta 有 26%/90% 的分割点能过。

**一行权限, 挡了一个能救命的发现三次。而系统每次都把它记成 `FAILED_SHIFT`。**

### 两个必须分开的东西

| | 是什么 | 解法在哪 | 加压有用吗 |
|---|---|---|---|
| **WALL** 墙 | agent 想干, 但**没有合法路径** | **人手里** (通常一行配置) | ❌ 加压只会让它更用力地撞墙 |
| **LAZY** 懒 | agent 有路径, 但没干 | **契约里** | ✅ |

**把两者记成同一件事, 系统就只会朝加压的方向走 —— 而墙永远不被修。**

### 尾声: 我修这堵墙的时候, 又犯了第 8 次

agent 写的解封行是**绝对路径**。我"顺手"改成了相对路径 `Bash(./.venv/bin/python research/:*)` ——
而班次的 cwd 是 **founder-os**, 那里既没有 `.venv` 也没有 `research/`。
**这条 allow 永远匹配不上任何真实命令。** 我在账本里把这堵墙标成了 `torn_down`。

墙还立着。配置显示"已放行"。账本显示"已解决"。**三层全绿, 全是假的。**

> **凡是要人照抄的配置, 就一定会被抄错。能生成的, 绝不让人抄。**

**装置** (全部机器强制, 非叮嘱):
- `src/walls.py` — 墙账本。撞墙**机器可读地登记** (散文没人读, 已证)。新墙必须带**一行可复制粘贴的解封动作**, 否则拒收 —— "一堵没有解法的墙, 对人来说只是一句抱怨。"
- **只有人能拆墙** (`FOUNDER_HUMAN=1`)。agent 自己宣布"墙拆了" = 自己给自己发赦免。
- `src/preflight.py` 门 3 — **拿真实命令去撞权限层**, 不问"配置里写了吗"。这道模拟咬住的正是上面那个 bug。
- `bin/shift` — `FAILED_SHIFT` 劈成 `FAILED_SHIFT/WALL` 与 `FAILED_SHIFT/LAZY`。
- `hooks/` — **权限层也归宪法管了**。能编辑 `.claude/settings.json` 的 agent 可以给自己加 `Bash(:*)` —— 那样所有的门 (包括那个 hook 自己) 都是纸做的。
- `install.sh` — 赛道权限从 `FOUNDER_TARGET` **自动生成**。

**拆除条件**: 当"人"不再是这个环里最慢的组件时。(即: 连续 10 堵墙在登记后 24h 内被拆。)

## 伤疤 #12 — **时钟装了, 但它永远不会响** (「声明≠有效」第 9 次)

**事故**: 用户问了一句 "ScheduleWakeup 是在 session 内部的吗?" —— 我去核实 OS 级时钟, 发现:

```
think:   无 StartCalendarInterval  →  **永远不会自己触发**   ← 出想法的班
build:   无 StartCalendarInterval  →  **永远不会自己触发**   ← 唯一往赛道交付的班
reflect: 周日 11:00 ✓
evolve:  周日 12:00 ✓
```

**这个号称"永不停转"的系统, 产出真东西的那两个班根本没有时钟。它一直是我推着走的。**

**根因** (install.sh):

```sh
for job in "think 9 30 *" "build 15 0 *" "reflect 11 0 0" "evolve 12 0 0"; do
    set -- $job          # ← $job **不带引号** → shell 把 `*` 通配展开成当前目录的文件名
```

`*` 被 glob 成文件名 → 写进 `<integer>` 标签 → **plist 语法损坏 → 无触发器**。
`reflect`/`evolve` 的 DOW=`0` 不含 `*`, **躲过一劫** —— 于是**半个时钟在走, 看起来一切正常**。

**为什么没人发现**:
- `launchctl list` 照样列出它们, **exit=0, 一片绿**
- `install.sh` 照样打印 **"装好了。think 09:30 · build 15:00 (每日)"**
- 日志里确实有 think/build 的 PASS 记录 —— **但那全是我手动跑的**

> **一个不会响的闹钟, 比没有闹钟更糟 —— 它让你以为你会被叫醒。**

**这是伤疤 #10 的原样复发** (产品出厂即坏, 还宣布成功)。而 #10 的修复 (开箱自检) **只自检了契约, 没自检时钟**。
**我给系统的每一部分都装了"验它真的生效", 唯独漏了那个负责"让系统活着"的部分。**

**装置**:
- `set -f` 关掉通配展开 + 用 `-` 而非 `*` 表示"每日" (**彻底不让通配符进入这段代码**)
- **时钟的开箱自检**: `plutil -lint` 验语法 + `plutil -extract StartCalendarInterval.Hour` **验它真的有触发器**。
  任一不过 → **拒绝安装**, 不宣布成功。
- 安装输出改成打印**实际解析出来的触发时刻**, 而不是复述源码里的意图。

**拆除条件**: 永不。

**元教训**: 用户那句问题不是质疑, 是**探针**。
**每一次"这个真的在跑吗?"都该被当成一次审计触发 —— 而不是一个可以用记忆回答的问题。**

## 伤疤 #13 — **我把思考排进了日程表** (用户一句话推翻了整个班次分类法)

**事故**: 我刚修完时钟, 报告说"下次自触发思考是 19:40 / 明早 09:30"。用户回:

> **"你告诉我, 你下次自己触发思考会是啥时候。为啥不是现在。"**

我承认了、当场想了。然后他说了更狠的一句:

> **"不是。。。我不是说我现在让你想你就现在想。。。我不告诉你你也应该一直想啊。"**

**两次都对, 而第二次打到了根。**

### 错在结构, 不在态度

**把 `think` 做成一个独立的、排期的班次, 在结构上等于宣布: 其他所有班次都可以不思考。**
而"排期的思考"本身就是打工人的定义 —— **想法成了任务, 不是本能。**

亲证 (同一小时内两次):
1. 我给思考排了个 25 分钟后的班次, 然后**收工**。
2. 用户点破后, 我"现在想" —— **但那还是因为他叫我想。**

这个系统的整篇 README 都在讲"动机不可能靠 prompt 说出来, 只能靠结构造出来" ——
**而我把思考本身留在了 prompt 层。**

### 物理约束 (必须诚实)

模型在两个回合之间**不存在**。"一直想"不可能靠意志实现 —— 没有"之间"。
能实现的只有一件事: **让思考成为每一次动作的税。**

### 装置 — **思考税** (`src/contract.py`)

> **除 `watch` 外, 每一个班都必须留下 ledger 变化。**
> 干完活却什么都没学到 = `FAILED_SHIFT`。

  · 不是"想法班想, 干活班干" —— 而是**干活的人必须带着一个新假设、一个死刑判决、或一次升级走出来**。
  · 契约仍**物理可满足** (伤疤 #4): WIP 满了也交得起税 ——
    **杀掉一个想法, 和提出一个想法, 都是思考的证据。**
  · `watch` 是唯一免税班: 它的活就是"确认无事发生"。

**拆除条件**: 当模型能在回合之间持续运行时 —— 那时"一直想"才是物理上可能的, 税才可以退。

> **你不能给"一直思考"排期。你只能让"不思考"交不了差。**
