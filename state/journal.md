### 发现: drive 箭头不存在 (车没接上赛道)

CLAUDE.md 和 GOAL.md 都画了这个闭环:

```
founder-os (车)  ──drive──►  trading (赛道)
      ▲                            │
      └──────  选择压力  ◄─────────┘
```

**上面那根 `drive` 箭头, 代码里不存在。** 证据全在本仓库内, 不需要读赛道:

- `FOUNDER_TARGET` 在 `shifts/*.md` 五个班次 prompt 里出现 **0 次**。它只活在 `bin/shift` (导出环境变量) 和 `bin/fitness-driven-project` (读数)。
- `src/contract.py` 的 `_git_head()` 用 `cwd=ROOT` —— 执行班的契约, **只要 founder-os 自己有 commit 就算达标, 永远不检查赛道**。
- 于是: 每个班都在改这辆车, 没有任何一个班会碰赛道。

车架在举升机上空转, 读着**别人赛道的秒表**。`state/fitness.jsonl` 里的 SHIPPED=3 / MONEY=8312.22 是交易项目自己跑出来的, founder-os 一行没贡献 —— 却记进了自己的适应度。

### 为什么这比"没用"更糟

它是一个**会骗人的适应度信号**。进化班读 fitness 趋势决定变异留不留, 而 fitness 的涨跌与变异**因果无关**。后果:
- 假阳性: 把赛道的自然进展记功给恰好在跑的那次变异 → 垃圾变异被固化
- 假阴性: 市场回撤时回滚掉好变异 → 好机制被随机杀死

**一个被混淆的控制器, 比没有控制器更危险** —— 没有控制器只是不进化; 混淆的控制器会朝噪声进化。

宪法花了大力气保证这个数**不可伪造**。它从没检查过这个数**是不是我们造成的**。
> **不可伪造 ≠ 可归因。** 这是 GOAL §2 的盲点。

**2. 反刍 / 该杀什么** — 杀掉 idea #2 (reflect 与 think 是否该合并)。理由不是它不重要, 而是它**在 drive 箭头存在前根本不可回答**: 没有班次写赛道 → 所有班次对赛道的贡献都是 0 → 比较两个班的毕业率 = 比较两个 0。且其证伪判据要"跑 20 班看统计" = 月级, 违反"天级证伪"纪律。这是最纯的 SCARS #3: **在一辆没接上赛道的车里优化组织架构图。**

**3. 十倍** — 若 SHIPPED 必须 3 → 30, 我不会"把 founder-os 想得更好"。我会让 **ledger 里躺着 30 条交易假设, 而不是 2 条关于班次分类学的元想法**。现在的 idea ledger 是一本**关于车的日记**。它应该是一本**关于赛道的赌注簿**。founder-os 的价值是机制; 它的产出是**上线的策略**。(与发现 #1 同根 —— 证据收敛。)

**4. 证伪 (最大的一注若是错的, 哪先露马脚)** — 最大的一注是"自举本身"(GOAL §4)。若自举是错的, 它**首先退化成自恋**: 无限递归地改进那个改进者。马脚是可日测的 —— `founder-os 产出的 commit 落在赛道仓库的比例`。连续 N 天 = 0, 自举即以当前形态失败。→ 落成想法 #4。

### ledger 自身的一个缺陷 (记下, 不动手)

idea #1 (自举) 的证伪判据是"连续 3 个思考班产不出非平凡想法"—— 这是个**努力指标**, 正是 GOAL §2.1 警告的陷阱 ("别测卖力, 测结果")。但**事后改判据 = 违宪**, 所以我不动它: 让 #1 按原判据受审, 由新的 #4 提供结果形态的判据。

暴露出的机制缺口: **ledger 没有合法的判据修订路径** —— 要么带着坏判据走到底, 要么违宪。这该由进化班处理 (例如: 允许 supersede, 但必须新开一条 id 并保留旧条的裁决)。

## 2026-07-12 · 思考班 #2

### 发现: preflight 是个安慰剂 —— 它给自己该拦的那堵墙开了绿灯

`src/preflight.py` 昨天 17:05 刚建, 为的是根治伤疤 #4 (不可满足的契约 = 墙)。
`state/shifts.log` 里同时留着这两行:

```
[think] === 开班 (契约可满足 (工具已放行: ['Bash(python3 src/ledger.py'])) ===
Ignoring 21 permissions.allow entries: this workspace has not been trusted.
```

**同一时刻**: preflight 说"绿灯, 工具已放行", 而运行时正在把**整个 allowlist 丢掉**。
它把班次一路放进了它本该拦住的那堵墙。**我这一班就是撞在上面死的** —— 和 #1 一模一样。

### 为什么这不是个小 bug: 同一个错误的第三次复发

preflight v1 检查的是**声明** (settings.json 里写了吗), 不是**有效能力** (运行时真生效了吗)。
这不是新错误, 是同一个模式第三次出现:

| | 检查了什么 | 漏了什么 |
|---|---|---|
| SCARS #1 | 规矩**写在文档里** | 规矩**被遵守**了吗 |
| GOAL §2  | 数字**不可伪造**   | 数字**可归因于我**吗 |
| SCARS #5 | 权限**写在配置里** | 权限**真的生效**了吗 |

三次都是: **验证了"声明", 当成了"现实"。**

**未生效的机制 = 不存在的机制。而一个报告"已生效"的未生效机制, 比没有机制更糟 —— 它让所有人停止追查。**

### 更狠的一层: 系统分不清"墙"和"懒"

我这班会被判 FAILED_SHIFT。**判得对** —— 但记录会写成"未提新想法"(= 懒)。
真因是: **我被封口了**(= 墙)。

这两者的正确处置**完全相反**:
- 懒 → 加压 (契约更严)
- 墙 → **减压 + 修工具** (加压只会让 agent 更快退化成打工人 —— 伤疤 #4 第二层)

**一个把"被封口"记成"偷懒"的系统, 会朝着加压的方向走, 而墙永远不会被修。**
这是 failed_shifts.log 当前的语义缺陷。

### 四问

**1. 不对称** — 见新想法 (下)。核心: 别人的 agent 框架永远是**有人在旁边点头**的 demo。
"声明的权限"和"有效的权限"之间那道裂缝, 在交互模式下**根本不存在** ——
所以他们撞不到, 也就不会去修。**只有真的把自己关进无人值守的契约循环、并且输给自己的人, 才看得见它。**

**2. 反刍 / 该杀什么** — 杀 #2 (reflect/think 是否该合并)。
在 drive 箭头存在前**不可回答**: 没有班次写赛道 → 各班对赛道贡献恒为 0 → 比较毕业率 = 比较两个 0。
且判据要"跑 20 班看统计" = 月级, 违反天级证伪纪律。最纯的 SCARS #3: **在一辆没接上赛道的车里优化组织架构图。**

**3. 十倍** — 我**不**采纳上一班提的"归因门"。
上一班诊断对了 (drive 箭头不存在), 但它开的两个药方 —— 归因门 + 自恋比率 —— **两个都是仪表**。
仪表只会告诉你箭头不存在; 它不制造箭头。**在一辆没接上赛道的车上再装两块表, 正是 SCARS #3 的第 11 个采集器。**
十倍不来自更好的测量, 来自**真的接上那根线**: build 班以 `cwd=$FOUNDER_TARGET` 跑, contract 检查**赛道**的 HEAD。
所以我把"自恋比率"降级成 drive 想法的**证伪判据**(它本来就该是判据, 不是独立项目), 归因门直接不提。

**4. 证伪** — 我最信的信念: "机制不衰减, prose 才衰减"(SCARS #1 的推论)。
如果它错了, 会先在哪露马脚? **就在今天**: preflight 是个机制, 它没衰减 —— 它**从一开始就没连上现实**。
机制不会衰减, 但**机制会验错东西**。一个验错东西的机制, 和一条被无视的规矩, 后果一样 —— 而且它还发绿灯。
→ 推论: 机制本身需要**有效性检验**, 不然它只是穿着代码外衣的 prose。
