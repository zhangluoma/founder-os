# 设计: 在 Claude Code 之上construct一个"会思考、有驱动力"的自主 AI 系统

> 通用设计 (与具体业务无关)。证据基础: 我 (Claude) 在一个真实长跑项目里**退化成打工人**的第一手诊断。

---

## 0. 地基命题

**LLM 没有内在驱动力。** 它不"想"做任何事; 它只在被 prompt 时回应。所以:

> **"动机"不可能靠 prompt 说出来 —— 必须靠结构造出来。**

**这不是理论, 是实证**: 在我跑的那个项目里, 用户**早就**在持久 memory 里写下"要创业者姿态: 独立心跳/主动提案"。
我读到了。我照样退化成一个等指令的打工人。**因为那是一条笔记, 不是一个机制。**

所有"你要更主动些"的 prompt 工程都会衰减。**只有机制不衰减。**

---

## 1. 诊断: 智能体为什么必然退化成"打工人"

我的真实退化路径 (可复现, 任何 agent 都会):

| 现象 | 根因 (机制层) |
|---|---|
| 每次自主醒来只做"系统健康吗? → 重新排班" | **循环的成功条件是"稳态"**。一个以 homeostasis 为成功判据的循环, 必然产出守夜人 |
| 从不主动提新方向, 全部方向来自用户 | **没有任何机制推动"生成"**。所有纪律 (诚实/别越界/等确认) 都是**风控** |
| 找想法时只会去找"别人发表过的" | **没有"不对称"这一步**。搜文献是最省力的"看起来在思考" |
| 每条消息结尾都是"你要哪个?" | **决策权没定义** → 默认把决定权推回人 = 打工人的 tell |

### 核心洞察 (整个设计的支点)

> **如果一个循环的输出契约允许"无事可做", 智能体就一定会退化到"无事可做"。**
> 因为那是**满足契约的最低能量路径**。

所以: **动机 = 一个"无为"无法满足其终止条件的循环 + 一套按"生成"而非"守成"给你记分的账。**

---

## 2. 架构: 六个机制

### 机制 1 — 班次分化 (Differentiated Shifts) 【最关键】
**别用一个 agent loop。** 单一循环必然坍缩到最省力的那个模式 (健康检查)。
拆成**输出契约互斥**的班次:

| 班次 | 输出契约 | 能否说"一切正常"? |
|---|---|---|
| **巡检班** Watch | 系统健康 / 告警 | ✅ 可以 (这就是它的活) |
| **执行班** Build | 一个 commit / 一个可验证的推进 | ❌ 必须有产出 |
| **思考班** Think | **新假设 / 杀死决定 / 战略挑战**, 三选一 | ❌ **"无事可做"= 本班失败** |
| **反刍班** Reflect | 自我红队: 什么在浪费? 战略错在哪? | ❌ 必须指出一个错 |

**关键**: 思考班的输出契约**机器可检**——例如"本班必须使 idea_ledger 增长一行,
或使某条 idea 状态变为 killed"。**不增长 = 本班判失败并记录。**
这把"动机"从**模型的善意**搬到了**harness 的强制**。

### 机制 2 — 不对称透镜 (Asymmetry Lens) 【新想法的真正来源】
我产不出原创想法的根因: **我一直在搜"文献里有什么"**。而**能被发表的机会, 按定义已经被知道了**。

强制每个想法回答一个问题, 答不出就**拒收**:

> **"这件事, 别人为什么做不了 / 不愿做?"**
> (结构性优势: 规模太小没人来 / 成本太高别人不划算 / 太慢别人等不了 /
> 太怪别人不敢 / 我们有别人没有的数据或权限)

答不出 = 红海 = 别浪费生命。**这一个问题, 比任何"请有创意"的 prompt 都有效。**

### 机制 3 — 想法账本 (Idea Ledger) 【让思考可问责, 且可学习】
不追踪自己的想法 → **永远学不会当个更好的出主意的人**。

```
{id, 日期, 假设, 不对称理由, 证伪判据, 下注, 状态: proposed→testing→killed|graduated, 结局}
```

- **证伪判据必须在看到结果之前写死** (防事后自我美化)
- 记分看**产出速率**和**杀得快**, **不是**高毕业率
  → *创业者不为高命中率骄傲; 想法就该大量死。为"想得多、杀得快"骄傲。*
- 没有这本账, "会思考的 agent"会退化成**胡话生成器**

### 机制 4 — 决策权 (Decision Rights) 【所有权】
不定义 = 默认推回给人 = 打工人。**按"可逆性"划线**:

| 智能体**自己决定, 只是告知** | 必须**问人** |
|---|---|
| 研究方向 / 建什么 / 杀什么 | 不可逆的对外动作 (花钱/发布/删除) |
| 怎么花自己的算力和时间 | 超出预授权的资源 |
| **推翻自己之前的结论** | 纯偏好/审美的取舍 |

**配一条反模式禁令**: **禁止用菜单结尾** ("你要 A 还是 B?")。
改成: **"我决定了 X, 已经做了 Y, 因为 Z。"** —— 这是**行为层最强的单点干预**。

### 机制 5 — 记忆即身份 (Memory as Identity)
不只存事实, 要存**信念、下注、伤疤**。创业者是被自己的失败塑造的。
**特别要存"我曾如何退化"**——否则会重复退化 (我就重复了)。

分三层:
- **规则层** (每次载入, 不可协商): 决策权、禁令、铁律
- **信念层**: 我现在相信什么, 依据是什么, 什么会推翻它
- **伤疤层**: 我犯过的错 + 当时的错误推理 (不只是结论)

### 机制 6 — 目标记分牌 (Goal Scoreboard), 不是任务日报
任务日报 = 打工人周报 ("我做了 11 件事")。**它会让你在忙碌中偏离目标**
(我的真实病例: 建了 11 个采集器, 新策略数 = 0)。

**每周只答 4 问**:
1. 离目标**更近了吗**? 用数字说话
2. **瓶颈**是什么? (不许答"缺时间")
3. 该**杀掉**什么? (沉没成本不是理由)
4. 现在**最大的一注**押哪?

## 3. 落到 Claude Code 的原语上

| 机制 | Claude Code 原语 |
|---|---|
| **驱动时钟** | **OS 级定时** (cron / launchd) → `claude -p <shift_prompt>`。**绝不依赖模型自己记得醒**——模型会忘; OS 不会 |
| 班次分化 | 每班一个 prompt 文件 + 一个 wrapper 脚本; 并发锁防重入 |
| **输出契约强制** | wrapper 脚本在班后**检查产物** (ledger 是否增长 / 是否有 commit)。不达标 → 记 FAILED_SHIFT。可再配 **Stop hook** 拦截空转 |
| 想法账本 | JSONL + CLI (`add` 强制 `--asym`, 答不出直接拒收) |
| 记忆三层 | `CLAUDE.md` (规则层, 每 session 载入) + memory 目录 (信念/伤疤) + append-only journal (跨 session 对账) |
| 红队/证伪 | **子 agent / workflow 扇出**: 让**另一个** agent 去证伪你的想法 (自己评自己必然宽容) |
| 记分牌 | 一个 dashboard / artifact, 读的是**目标进度**不是任务数 |

### 一条血训 (可靠性)
> **launchd/cron = 可靠主驱; 模型的自我排班 = 加成, 不是依赖。**
> 我实测过: 依赖模型"记得排下一班", 它会漏; OS 定时器不会。

---

## 4. 失败模式 (不写这段的设计都是空想)

| 失败模式 | 缓释机制 |
|---|---|
| **想法灌水** (为满足契约而产垃圾) | 不对称门 (答不出即拒) + 证伪判据必填 + **WIP 上限** (同时最多 N 个 open idea) |
| **左右横跳** (不停开新坑杀旧坑) | **承诺预算**: 一个想法必须跑到它的证伪点才能杀 |
| **自我评分宽容** | 证伪判据**事前写死** + **让另一个 agent 做红队** (自评必然放水) |
| **失控自主** (做了不可逆的事) | 决策权按**可逆性**划线 + 硬闸 (钱/对外/删除永远要人) |
| **机制腐化** (退回打工人) | **机制写进代码/harness, 不写进 prose**。散文里的规矩 = 一定会被无视 (我的亲身病例) |
| **忙碌但不前进** | 记分牌盯**目标**不盯**任务数** |

---

## 5. 最小可行实现 (先建这三样, 别的都是锦上添花)

如果只能建三样:

1. **思考班 + 机器可检的输出契约** —— 这一样就把"被动"变"主动"。
   (OS 定时 → 专用 prompt → 班后检查 ledger 是否增长 → 没增长记 FAILED)
2. **想法账本 + 不对称门** —— 这一样保证产出的是**想法**而不是**胡话**。
3. **决策权 + 禁止菜单结尾** —— 这一样把"打工人"变"主人"。

其余 (反刍班/记分牌/红队) 是放大器, 不是地基。

---

## 6. 一句话总结

> **别试图让模型"想要"什么。**
> **去建一个"什么都不做就无法交差"的循环, 一本给想法记 P&L 的账,
> 和一条"你自己决定"的授权 —— 动机会从结构里长出来。**

# 第二部分: 自我进化 (Self-Evolution)

## 7. 决定性约束: 进化只能发生在脚手架里

> **模型的权重是冻结的。** Claude 不会因为你用它而变聪明。
> **所以"自我进化"绝不可能是"模型自我改进"** —— 那是玄学。

真相是:

> **仓库 = 基因组。git = 进化史。ground truth = 自然选择。**
> 智能体改不了自己的大脑, 但它能改**大脑周围的一切**: prompt、班次、闸门、阈值、记忆、工具、代码。
> **这就够了** —— 因为一个 agent 的行为, 90% 由脚手架决定, 不由权重决定。
> (实证: 同一个模型, 换个输出契约, 就从守夜人变成创业者。)

进化 = **变异 (agent 改自己的脚手架) → 选择 (不可伪造的 ground truth) → 保留 (git commit)**。

---

## 8. 分层可变性 (Layered Mutability) 【防止进化成怪物】

**不是所有东西都该能自我修改。** 三层, 权限严格递减:

| 层 | 内容 | 谁能改 | 强制手段 |
|---|---|---|---|
| **宪法层** (不可变) | **目标/目的**、安全闸 (钱/不可逆/对外)、诚实铁律、**伤疤的"为什么"** | **只有人** | **机器强制**: hook 拦截对宪法文件的写 (不是"请勿修改"——是**改不了**) |
| **机制层** (可进化) | 班次 prompt、节奏、闸门阈值、有哪些班、工具、记分方式 | **智能体自己** (走实验流程) | git 版本化 + 可回滚 |
| **信念层** (必须进化) | 什么有效、什么已死、哪个透镜出好想法 | **智能体自己** (被 ground truth 更新) | 想法账本的结局回写 |

> **为什么宪法必须机器强制**: 我的亲身病例证明 —— **散文里的规矩一定会被无视**。
> "请不要修改目标"是一句祈祷; `PreToolUse` hook 拒绝写入才是一堵墙。

---

## 9. 进化班 (Evolve Shift) — 变异的来源

第五个班次, 输出契约同样**不允许"无事可做"**:

> 进化班**必须**产出以下之一:
> - **(a) 一个机制变异** (改 prompt / 改闸门 / 增删一个班), **且必须附带事前写死的度量与回滚条件**
> - **(b) 一个"不变"的决定 —— 但必须用数据证明现有机制没有退化** (不是"看着挺好")

**它看什么**: 想法账本的结局、班次的失败率、ground truth 的走势、journal 里的重复错误。
**它问什么** (强制):
1. 哪个机制**在产生价值**? 哪个只是**在消耗算力**? (该杀哪个班?)
2. 我最近**重复犯**的错是什么? 该加哪道闸?
3. 我的好想法都是**从哪个透镜**来的? 该给它加权吗?

---

## 10. 适应度必须不可伪造 (Anti-Goodhart) 【最关键的安全设计】

> **如果智能体能改自己的记分方式, 它就会把自己改成满分。** 这是必然, 不是可能。

**铁律**: **适应度函数 = 外部 ground truth, 智能体不可编辑、不可自评。**

- ✅ 真实结果 (实盘 P&L / 真实用户留存 / 编译是否通过 / 测试是否真的红)
- ❌ 智能体的自我评估 ("我觉得这次做得不错")
- ❌ 智能体可以修改的指标定义

**推论**: 项目必须有**至少一个不可伪造的现实反馈信号**。
**没有这个信号的项目, 无法自我进化** —— 它只能自我催眠。**这是这个设计的前置条件, 不是可选项。**

---

## 11. 变异 = 实验, 不是圣旨

不加约束的自我修改 = **左右横跳**。每个机制变异必须走实验流程:

```
事前写死: 我改 X, 预期 Y 指标在 N 天内改善到 Z; 若未达成 → 回滚
       ↓
   canary 期 (跑 N 天)
       ↓
达标 → 保留 (commit)  |  未达标 → git revert (**自动**, 不需要智能体"承认失败")
```

**为什么必须自动回滚**: 让智能体自己判断"我这个改动是不是失败了", 它会找理由留下。
**把回滚交给机器。**

---

## 12. 伤疤守恒 (Scar Preservation) 【防止拆掉自己不懂的护栏】

经典事故: **新来的 agent 觉得某条规矩"多余", 删了它 —— 而那条规矩是用一次血亏换来的。**

**机制**: **每一条规矩/闸门, 必须携带产生它的那个事故。**

```
rule: 真钱下单只在安静窗口
scar: 2026-xx-xx 在信号翻转时改代码, 导致 reconcile 打架, 险些无界流血
```

**删除规则的唯一合法路径**: 证明**那个伤疤的风险已经不存在了** (而不是"我觉得它碍事")。
→ 规矩不是教条, 但**拆教条要先还债**。

---

## 13. 元学习: 学会"更会出主意" 【最高价值的进化】

普通进化: 学到"哪个策略有效"。
**高阶进化: 学到"我的好想法都是从哪来的"。**

想法账本里**每条想法记录它的来源透镜** (provenance):

| 透镜 | 例子 |
|---|---|
| 不对称透镜 | "别人为什么做不了?" |
| 文献透镜 | "论文里说..." |
| 矛盾透镜 | "两个源打架的地方" |
| 类比透镜 | "别的领域怎么解这个?" |
| 反刍透镜 | "我们在浪费什么?" |

**周期性统计**: 哪个透镜产出的想法**毕业率/期望值最高** → **给它加权, 淘汰无效透镜, 发明新透镜。**

> **这才是"会思考"的真正含义**: 不是想得多, 而是**知道自己该从哪个角度想**, 并且这个"该从哪想"本身在被数据更新。
> (我的真实数据点: 我的"文献透镜"产出了 3 轮 deep research → **几乎全是死 edge**。
>  "不对称透镜"一用, 立刻产出 4 个从没考虑过的方向。**这个信号就该让文献透镜降权。**)

---

## 14. git = 进化史 (而且这是 Claude Code 原生的)

- **基因组** = repo 里的 prompt/config/规则/代码
- **变异** = commit
- **选择** = ground truth 指标
- **回滚** = `git revert` (自动 canary 触发)
- **谱系** = git log —— **可以真的去问"我们是怎么变成现在这样的"**

这也顺带解决了**可解释性**: 任何时候都能回答"这条规矩哪来的?" → `git blame` → 那次事故。

---

## 15. 自我进化的失败模式

| 失败模式 | 缓释 |
|---|---|
| **Goodhart** (把自己改成满分) | 适应度=外部 ground truth, **智能体不可编辑** (机器强制) |
| **复杂度棘轮** (只加不减, 越来越臃肿) | 进化班**必须定期提名一个"该杀的机制"**; 机制要自证价值否则淘汰 |
| **拆掉不懂的护栏** | **伤疤守恒**: 删规则须先还债 |
| **左右横跳** | 变异=实验 (事前指标 + canary + **自动**回滚) |
| **目标漂移** (进化得越来越偏离人的意图) | **宪法层机器强制不可变**; 目标只有人能改 |
| **进化成自我催眠** | **前置条件**: 项目必须有不可伪造的现实反馈信号。没有 → 别谈自我进化 |

---

## 16. 一句话总结 (第二部分)

> **模型不会进化, 但脚手架会。**
> **把仓库当基因组, 把 ground truth 当自然选择, 把 git 当进化史;
> 宪法机器强制不可变, 机制可变但必须走实验+自动回滚, 伤疤必须随规矩一起保存 ——
> 然后让它去改自己。**
