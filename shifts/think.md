## 输出契约 (硬性 — 机器会检)

你**必须**产出以下之一, 并落进 idea ledger:

    python3 src/ledger.py add "假设" --asymmetry "别人为什么做不了" --falsify "什么会杀死它" --lens asymmetry
    python3 src/ledger.py set <id> --status killed --outcome "为什么死"
    python3 src/ledger.py set <id> --status testing|graduated --outcome "..."

**"一切正常 / 无事可做" = 本班失败。** 那是巡检班的活, 不是你的。
班后 `src/contract.py` 会检查 ledger 是否真的变了。没变 → FAILED_SHIFT, 记录在案。

## 强制自问 (真答, 不许走过场)

**1. 不对称** — 我要提的这件事, **别人为什么做不了 / 不愿做?**
   (规模太小没人来 / 太慢别人等不了 / 太怪别人不敢 / 我们有别人没有的数据或权限)
   **答不出 = 红海 = 别提。**

   ⚠️ **警告**: 搜"文献/别人怎么做的"是**最省力的伪思考**。
   **能被发表的机会, 按定义已经被所有人知道了。** 真正的想法来自不对称, 不来自综述。

**2. 反刍** — 我们最近做的哪件事**其实是浪费**? 该**杀掉**什么?
   (沉没成本不是理由。见 constitution/SCARS.md #3: 忙碌 ≠ 前进)

**3. 十倍** — 如果必须让结果 **10 倍**, 我会做什么**完全不同**的事?

**4. 证伪** — 我最相信的那个信念, 如果是错的, **会先在哪露出马脚**?
